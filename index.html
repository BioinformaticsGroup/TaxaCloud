Description
The ever-increasing demand for high-throughput metagenomic data analysis has led to an unsustainable demand for computational training and resources. Elastic computing (EC) is a service oriented computing mode that computing resources can be scaled up and down easily by the cloud service provider, such as Nectar, and Amazon Web Services (AWS). We provide a visual web application platform with a Web 2.0 user interface and an automatic scaling analysing service to support wet-lab staffs for metagenomic data analysis. This platform can request and startup virtual machine resource after that researchers submit metagenomic data for analysing by using automatic scaling analysing service.
 
This platform has been designed to be an easy-to-use web application framework for the organisation and analysis of high-throughput metagenomic datus on bacterial 16S rRNA sequencing datasets and fungal ITS sequence datasets. It integrates QIIME (Quantitative Insights Into Microbial Ecology), a set of bioinformatics software packages and scripts, and an automatic scaling analysing service, with interactive visualisations (ownCloud web interface) to enable researchers to receive results from demultiplexing and quality filtering, OUT picking, taxonomic assignment, and diversity analysis with submitted sequencing data. These results can be used to do metagnomic analysis for microbial communities.

Prerequisites
This system provides service with automatic workflow for uploading Illimina fastq data and using QIIME image to do microbial community analysis using QIIME, therefore, this system only accepts gzip-compressed (e.g., .fastq.gz) or uncompressed (e.g. .fastq) fastq files. For more details, please look through qiime.org/tutorials/processing_illumina_data.html.
The format of paired-end fastq files should start with unique ID  following with separator '_' and 'R1' or 'R2', e.g. MPR3AA_R1.fastq(.gz) or MPR3AA_R2.fastq(.gz).
